# 01_DATABASES — XRM 全球数据库中心

## 用途
存储 XRM Intelligence 所有结构化业务数据库。
每个子目录是一个独立数据库，长期持续积累，是整个系统的核心资产。

## 子目录结构

| 目录 | 内容 | 主要字段 |
|------|------|----------|
| `suppliers/` | 全球供应商数据库（中国、东南亚、中东） | 供应商名称、国家、品类、联系方式、认证 |
| `hotel-projects/` | 全球酒店 FF&E / OS&E 项目数据库 | 项目名称、地点、星级、采购阶段、预算范围 |
| `ai-infrastructure/` | AI 基础设施 / 数据中心供应商数据库 | 产品类型、技术规格、合规认证、价格区间 |
| `epc-me-firms/` | EPC、M&E、建筑设计公司数据库 | 公司名、专业领域、已完成项目、所在地区 |

## 文件命名规则
```
YYYY-MM-DD-[来源]-[品类/地区].json   ← 结构化数据（AI 可读）
YYYY-MM-DD-[来源]-[品类/地区].md     ← 研究报告（人类可读）
```

## AI 自动更新协议
- 每次新增供应商：追加 JSON 条目，**不覆盖**原文件
- 每次更新信息：在原条目增加 `updated_at` 字段
- 新建月度快照：`YYYY-MM-suppliers-snapshot.json`
- Schema 定义见 `07_SYSTEM/schemas/`

## 注意
- 不存储客户名称、项目合同金额等保密信息
- 所有字段须符合对应 Schema 定义
